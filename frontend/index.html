How it works

Two-step AI pipeline using Claude Sonnet

  1. 1

    Document parsing

    The uploaded PDF or DOCX is converted to plain text using PyMuPDF / python-docx. No file is stored on disk.

  2. 2

    Structured clause extraction (Claude, Step 1)

    Claude uses forced tool use to extract contract type, parties, effective date, and every meaningful clause verbatim into a typed JSON schema, reliable and machine-readable.

  3. 3

    Playbook analysis & redlines (Claude, Step 2)

    The extracted clauses are compared against the playbook. Claude returns a risk level, recommended action, clause-by-clause status, and specific redline language for every deviation, delivered as structured JSON.

  4. 4

    Triage & routing

    Auto-approve: all clauses match, no human review needed. Fast-track: minor deviations only, quick 10-min check sufficient. Full review: meaningful deviations, careful human review needed. Escalate: high-severity issues, legal must review before responding.
